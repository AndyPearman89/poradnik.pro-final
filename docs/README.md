# Dokumentacja Poradnik.pro

Ten katalog zawiera skondensowaną dokumentację operacyjną i prompty wykonawcze dla systemu Poradnik.pro.

## Spis

1. `architecture/system-blueprint.md`  
   Definicja systemu, warstwy biznesowe, modele przepływów i cele monetyzacji.

2. `requirements/frontend-spec.md`  
   Pelna specyfikacja frontendowa dla nowego theme poradnik.pro i integracji z PearTree Core.

3. `implementation/roadmap-backlog.md`  
   Plan wdrożenia etapami (MVP -> skala), backlog techniczny, kryteria odbioru.

4. `prompts/master-prompts.md`  
   Znormalizowane prompty do pracy z agentami kodującymi (build, refactor, SEO, lead engine, QA).

5. `prompts/acceptance-checklist.md`  
   Checklisty odbiorowe dla funkcji krytycznych: konwersja, SEO, wydajność, mobile.

6. `architecture/monetization-blueprint.md`  
   Hybrydowy model przychodu: AdSense + afiliacja + lead engine, wraz z segmentacja intencji i KPI.

7. `architecture/system-1-2-3-monetization.md`  
   Operacyjny model 1/2/3: szczegolowe role AdSense, afiliacji i lead engine, mix stron, KPI, plan 60 dni i symulacja przychodu.

8. `architecture/full-system-enterprise.md`  
   Kompletny model enterprise obejmujacy segmentacje intencji, architekture stron, automatyzacje, kanaly ruchu, metryki i plan wdrozenia.

9. `architecture/seo-system-enterprise.md`  
   Kompletny system SEO enterprise: filary, programmatic SEO, technical SEO, intent routing, plan 0-90 dni i metryki wzrostu.

10. `architecture/complete-system-enterprise.md`  
   Finalny model operacyjny SEO + Content + Intent + AdSense + Affiliate + Lead z planem 0-90 dni i modelem przychodu.

11. `prompts/poradnik-pro-full-autonomous-system-builder.md`  
   Pelny prompt wykonawczy do autonomicznego budowania systemu Poradnik.pro (frontend, SEO, monetyzacja, automatyzacja).
